- `az login`の実施

## 1.IaCコードの利用

### 1-1. IaCコードの分析

Bicepなどは都度更新がされているためMCPサーバーを利用しながら分析を実施したほうが良いです。
現在のIaCコードの内容をベースに人間が確認可能なドキュメントの情報に落とし込みを行い人間がレビューできる状態にします。

```markdown
`infra`ディレクトリ内にあるIaCコードを参照し現在のAzure構成についてドキュメントを作成してください。
Azureのリソース間の連携については、AAやMermaidを使用して図を作成してください。
Bicepの機能やプロパティについては、最新のMS Learnの情報を参考にする必要があります。
```

GitHub Actionsを使用するため、デプロイに必要な環境変数を把握しGitHubに設定するための情報を収集します。

```markdown
GitHubのActionsを使用してAzureリソースの展開を行います。
環境変数に必要な設定を洗い出してください。
```

からの…

```markdown
{{設定を行ってもらう}}
```

大量のリソースがある場合は、リソースの種類や構成を把握するために、リソースの種類ごとにドキュメントを作成することも検討したほうが良いかもしれません。
また、サブエージェントを利用したカスタムエージェントの作成や手順なんかを記載したSkillを作成しても良いかもしれません。

### 1-2. 要件からAzureリソースを利用する相談

アプリケーションの要件から必要なAzureリソースを提案することができるか検討します。
他のコンテキスト読まないようにそのドキュメントがあるディレクトリで実行したほうが良さそうです。
`docs/sample`ディレクトリ内のドキュメントを参照しながら実行してください。

```markdown
`init.md`に記載されているアプリケーションの要件を満たすために必要なAzureリソースについて相談を行います。
要件を満たすために必要なAzureリソースの種類や構成について提案を行ってください。
最新のMS Learnの情報を参考にして構成を提案してください。
不明瞭なもの、曖昧なものがある場合は、必要に応じて質問を行いながら提案を行ってください。
```

### 1-3. 推奨構成をベースにしたアセスメント

現在の構成とMS Learnなどで推奨されている構成を比較し、改善点やリスクについてのアセスメントを行います。

```markdown
現在構成されているAzureインフラの改善点を確認します。
Azureアーキテクチャセンターや、クラウド設計パターンを参照しながら、現在の構成と推奨されている構成を比較し改善点を提示してください。
参照したAzureのリソースのドキュメントや、推奨されている構成のドキュメントを提示してください。
不明瞭なもの、曖昧なものがある場合は、必要に応じて質問を行いながら日本語で提案を行ってください。
```

## 2. GitHubの利用（Deploy）

### 2-1. Actionsで発生したエラーを補足し修正する（デモ）

`resources/github-actions-error` ディレクトリにGitHub Actionsで発生したエラーの情報を格納しているため、そちらを参照して作業を行ってください。
GitHub Actionsで発生したエラーの内容を解析し、修正の提案を行います。
**実際のGitHub Actionsのエラーを用いての作業は難しいと思うので、GHコマンドを利用した動作はデモでの実施を想定しています。**

```markdown
`resources/github-actions-error`ディレクトリにGitHub Actionsで発生したエラーの情報が格納されているため、そちらを参照しエラーの原因を調査し報告してください。
```

### 2-2. 既存AzureリソースからのIaCコードへの反映

`resources/arm-resource` ディレクトリにAzureリソースの情報(ARMの情報)を格納しているため、そのディレクトリ上で作業を行ってください。

```markdown
`resources/arm-resource`ディレクトリにAzureリソースの情報(ARMの情報)が格納されているため、そちらを参照し指定されたAzureリソースをターゲットに`infra`にIaCコードを作成し、Azure構成についてドキュメントを作成してください。
Azureのリソース間の連携については、AAやMermaidを使用して図を作成してください。
Bicepの機能やプロパティについては、最新のMS Learnの情報を参考にする必要があります。
```

## 3.Azureの利用（Deploy）

### 3-1. azコマンドなどを利用したエラー情報の収集と分析

Azureリソースで発生しているエラーを抽出し解析します。
`resources/azure-error`に情報が格納されているため、そちらを参照して作業を行ってください。

```markdown
`resources/azure-error`ディレクトリにAzureで発生したエラー情報を格納しています。
このエラー情報を解析し、エラーの内容や原因を特定してください。
```

## 4. Azureの利用（運用と運用の改善）

### 4-1. Azureリソースとコスト削減の相談を実施する

Azureリソースのコスト削減のための相談を行います。
※CLIのリサーチツールを使う方が価格の精度は上がるかもしれません。

```markdown
`azure-resource` 内のAzureリソースの情報を参照し、コスト削減の検討を行ってください。
利用金額に関わる情報は `azure-usage`　に存在します。
コスト削減によって発生するメリットとデメリット、影響範囲、削減額の試算を提示する必要があります。
Azureの最新の価格情報はWebで確認する必要があります。
```

## 4-2.azコマンドなどを利用したエラー情報を用いた開発計画の作成

Application Insightsなどで収集したエラー情報をもとに、開発計画の作成を行います。
`resources/insights-error`に情報が格納されているため、そちらを参照して作業を行ってください。

```markdown
`resources/insights-error`ディレクトリにApplication Insightsなどで収集したエラー情報が格納されているため、そちらを参照しエラーの内容や原因を特定し、開発計画の作成を行ってください。
500エラーに関する修正のみを対象とします。
開発計画は `docs/fix-plan` ディレクトリにドキュメントとして作成してください。
```
