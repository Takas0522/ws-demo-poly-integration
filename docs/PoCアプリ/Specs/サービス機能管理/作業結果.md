## 検出された問題・所見

1. **servicesコンテナのパーティションキー変更**: 既存の`/id`から`/partitionKey`への変更が必要。Azure CosmosDBはPK変更不可のため、コンテナ再作成が必要。開発環境ではDevContainer再起動で対応可能。
2. **adminロールのクロステナント操作**: `update_tenant_service_feature`で`admin`ロールのユーザーが自テナント以外の機能設定を更新できる可能性がある。現状はバックエンドAPIの認証で防御されているが、将来的にはテナントID検証を追加することを推奨。
3. **BFF PUTルートの認可チェック**: BFF層では認証（getSession）のみで認可（ロールチェック）は行っておらず、バックエンドAPI側で実施。多層防御の観点からBFF層にもロールチェック追加を検討。

---

## 改善提案

1. **テナントID検証の強化**: `update_tenant_service_feature`にテナントID検証（JWT tenant_id == path tenant_id、ただし`global_admin`は除外）を追加
2. **E2Eテストの追加**: フロントエンドからバックエンドまでの統合的なE2Eテスト（Playwright）を追加し、UIフローの検証を自動化
3. **機能一覧のキャッシュ戦略改善**: 現在はローカルstateでキャッシュしているが、TanStack Queryを使用したキャッシュ管理への移行で、複数タブ間の整合性やバックグラウンド再検証を実現可能
4. **アクセシビリティ向上**: トグルボタンにrole="switch"属性の追加、機能テーブルのcaption要素追加を検討
