# 08 — フロントエンドUI実装

## 概要

サービス設定ページ（`/dashboard/services`）に、割り当て済みサービスの展開機能（アコーディオン）とサービス機能一覧＆有効/無効トグル UI を追加する。

## 背景・目的

管理者がブラウザから直感的にサービス機能の有効/無効を切り替えられる操作インターフェースを提供する。テナント選択→サービス展開→機能トグルの操作フローを既存のサービス設定ページに統合する。

## 対象リポジトリ

- `src/front/` — サービス設定ページの変更

## タスク詳細

### 実装内容

1. **`src/front/app/dashboard/services/page.tsx` の変更**

   既存のサービス一覧テーブルに以下の UI 要素を追加する:

   - **サービス行の展開ボタン（▶/▼アイコン）**
     - 割り当て済みサービスの行にのみ表示
     - クリックでアコーディオン形式の機能一覧セクションを展開/折りたたみ
   - **機能一覧テーブル**（展開時に表示）
     - テナント別機能設定 API（`GET /api/tenants/{id}/services/{serviceId}/features`）を呼び出して取得
     - 各行に `feature_name`、`description`、有効/無効トグル、`is_default` 表示を含める
   - **有効/無効トグル**
     - `is_enabled` の状態を反映（ON/OFF）
     - クリックで `PUT /api/tenants/{id}/services/{serviceId}/features/{featureId}` を呼び出し
     - 操作後のレスポンスで UI を即時更新（リスト再取得）
     - ローディング中はトグルを無効化してスピナーを表示
   - **権限制御**
     - 現在のユーザーセッション情報からロールを取得
     - `admin` / `global_admin` ロールを持たないユーザーはトグルを `disabled` で表示（読み取り専用）
   - **デフォルト/カスタム表示**
     - `is_default: true` の機能には「デフォルト」バッジを表示
     - `is_default: false` の機能には「カスタム設定」バッジを表示

2. **状態管理の追加**
   - `expandedService: string | null` — 展開中のサービス ID
   - `serviceFeatures: Map<string, TenantServiceFeature[]>` — サービスごとの機能一覧キャッシュ
   - `featureLoading: string | null` — 機能一覧ロード中のサービス ID
   - `togglingFeature: string | null` — トグル操作中の featureId

3. **エラーハンドリング**
   - 機能一覧取得失敗時にエラーメッセージを表示
   - トグル操作失敗時にエラーメッセージを表示し、UI の状態を戻す

## 備考

- トグルコンポーネントは TailwindCSS で実装する。外部UIライブラリの追加は不要
- サービス展開時の機能一覧取得は遅延ロード（展開クリック時に API 呼び出し）とする
- 一度取得した機能一覧はローカル state にキャッシュし、同一サービスの再展開時は再取得しない（トグル操作後は再取得する）
- ユーザーセッション情報からのロール取得は、既存の認証フロー（`getSession()` / Cookie 内 JWT）を使用する。ページコンポーネント内でユーザーロールを判定し、admin 権限の有無をトグルの disabled 属性に反映する
