# 02 — バックエンドモデルとスキーマ定義

## 概要

`ServiceFeature` と `TenantServiceFeature` の CosmosDB ドキュメントモデル（Pydantic）および API リクエスト/レスポンススキーマを定義する。

## 背景・目的

サービス機能管理の全レイヤー（リポジトリ → サービス → API）で共通利用されるデータ構造を先行して定義する。既存の `models/service.py` および `schemas/service.py` のパターンに準拠し、新規ファイルとして追加する。

## 対象リポジトリ

- `src/service-setting-service/` — モデルおよびスキーマファイルの新規作成

## タスク詳細

### 実装内容

1. **`src/service-setting-service/app/models/service_feature.py` の新規作成**
   - `ServiceFeature` クラス（Pydantic BaseModel）
     - フィールド: `id`, `type`（デフォルト `"service_feature"`）, `service_id`, `feature_key`, `feature_name`, `description`, `default_enabled`, `created_at`, `updated_at`（Optional）, `partitionKey`
   - `TenantServiceFeature` クラス（Pydantic BaseModel）
     - フィールド: `id`, `type`（デフォルト `"tenant_service_feature"`）, `tenant_id`, `service_id`, `feature_id`, `feature_key`, `is_enabled`, `updated_at`, `updated_by`, `partitionKey`
   - 既存の `models/service.py` のスタイル（`Field(...)` の利用、`description` 付き）に準拠する

2. **`src/service-setting-service/app/schemas/service_feature.py` の新規作成**
   - `ServiceFeatureResponse` クラス
     - フィールド: `id`, `service_id`, `feature_key`, `feature_name`, `description`, `default_enabled`, `created_at`
   - `TenantServiceFeatureResponse` クラス
     - フィールド: `feature_id`, `service_id`, `feature_key`, `feature_name`, `description`, `is_enabled`, `is_default`, `updated_at`（Optional）, `updated_by`（Optional）
   - `UpdateTenantServiceFeatureRequest` クラス
     - フィールド: `is_enabled`（bool、必須）
   - `ServiceFeaturesListResponse` クラス
     - フィールド: `service_id`, `features`（List[ServiceFeatureResponse]）
   - `TenantServiceFeaturesListResponse` クラス
     - フィールド: `tenant_id`, `service_id`, `features`（List[TenantServiceFeatureResponse]）
   - 既存の `schemas/service.py` のスタイルに準拠する
